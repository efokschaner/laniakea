Let webRtcDataChannelPayloadMTU

webRtcDataChannelPayloadMTU: number = 1131
  • Defined in network-peer/src/acking-peer.ts:46

This is the largest number of bytes we should send in a data channel payload to avoid fragmentation. It's not a guaranteed fact as MTU is not a constant. So this is a conservative estimate. See this for derivation: https://groups.google.com/d/msg/discuss-webrtc/LZsm-jbP0zA/JITEhtx4HAQJ
